偶然从小伙伴那发现hexo这么个玩意儿，小巧耐玩，感觉很赞，自己也能靠hexo搞个博客挂在git上，省心。这种文章写法和传统略微不同，使用的markdown格式，各种风骚的标题加分割线以及排版深得我心，就喜欢简单~

外加考虑到自个已经老大不小，在IT行业摸爬滚打积累的东西挺多，经常会第一时间无从思索，脑子一片空白。仔细想想，经验什么的，还是写下来才靠谱，记下所学的同时，另外也可以换种心情唠唠嗑。是该花点时间改改我这个不写博客的“优点”了。

怎么开始

正所谓万事开头难，也就先挂个第一篇（扯淡篇），刷个存在先。接下来写的也会比较杂，绝大部分是自己的即时经验所得，可以是关于游戏技术，或者是新发现，个人碰到的难题解决方案等。不管网上有没有，我得有，给自己提个醒，要是有幸能帮助他人也是极好的，还可以是自己学习某领域的探索过程，甚妙。

来个a+b

#include<cstdio>
#include<cstring>

int main()
{
       int a,b;
       while(scanf("%d%d",&a,&b)!=EOF)
       {
           __asm
           {
            mov EAX,a
            mov EBX,b
            add EAX,EBX
            mov a,EAX
        }
        printf("%d\n",a);
    }

    return 0;
}
